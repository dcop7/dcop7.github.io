## Dynamic categories (not in split DB — generated at runtime)

These pull from `data/countries.json` (250 countries) or the embedded banks /
Open Trivia DB API, so they already have large, low-repetition pools:

- **Bandeiras, Capitais, Continentes, Maior População** — 250 countries each.
- **Cultura Geral, Ciência, História, Desporto, Música, Animais, …** — embedded
  pt-PT banks (expanded earlier) with Open Trivia DB API fallback (EN).
- **Matemática, Tabuada, Relógios** — procedurally generated (effectively infinite).

## Image audit

- **284 questions carry images; 0 are broken** (validated against local files).
- Logos: **235** CC0 SVGs (Simple Icons) stored locally under `assets/icons/`.
- Monuments: **28** Wikipedia/Commons photos under `assets/monuments/` (~1 MB).
- Every image type falls back to a labelled placeholder if absent — no empty
  containers ever render.

## Weak categories (priority for future expansion)

Strongest-first, the categories still below a "large" bar are the **hand-authored
trivia** sets, all ~17–31 questions:

1. **sinais** (17) — limited by how many distinct PT road signs are worth drawing.
2. **f1** (21), **historia** (24), **ciencia** (24), **portugal** (24),
   **carros** (25), **tecnologia** (31).

These are **honestly not yet at the 100/250 targets.** They are pure-knowledge
trivia: reaching 250 *accurate, unique* entries each requires sourced authoring,
not generation, and was deliberately **not** padded with fabricated/placeholder
questions (per the project's no-fabrication rule). The generator approach used
for geography (322) and logos (235) is the template for scaling the rest from
verified sources.

## Tooling (`quizzes/tools/`)

- `validate.mjs` — fails the build if any question is malformed or references a
  missing local image. Run before shipping.
- `gen-logos.mjs`, `gen-monuments.mjs`, `gen-geografia.mjs`, `gen-simbolos.mjs`
  — regenerate categories from verified datasets/assets.
- `quizzes/assets/fetch-assets.mjs` — (re)download CC0 logos + Commons images.
